CULTURE GENERALE

Architecture 2 tiers ou 3 tiers

Serv 2 tiers : utilisateur échange des requête et réponse avec un serveur

Serv 3 tiers : l'utilisateur échange des requête et réponses avec un serveur. Ce serveur en revanche est lui même en échange avec un serveur de données (exemple pour les bases de données).

On parle de serveur applicatif pour le serveur qui reçoit les données directs de l'utilisateurs.

On parle de serveur de données pour... Un serveur qui stock et renvoie les données indirectement par l'utilisateur, et plus directement par le serveur applicatif.

Ce module est relié qu'à l'architecture 2 tiers pour l'instant.

Le modèle 3 tiers servira lorsqu'il faudra faire des pages interactives liées à des bases de données.

MODELE TCP/IP ET PROTOCLE HTTP

Modèle TCP/IP = modèle de représentation en 4 couches

Manière d'expliquer comment résoudre un problème complexe en réalisant un certains nombres d'étapes nécéssaires à la communications.

Schema sur documents du cours.

IP : Comparable a un numero de telephone.

Lorsqu'on entre un nom de domaine (nom pleinement qualifié), notre ordinateur fait la recherche de l'IP (le "numero" du site). S'il ne trouve rien, on ne peut juste pas acceder au site.

HTTP : HyperTextProtocol
TCP : Transmission Control Protocl
IP : Internet Protocol

METHODES

GET

Values contained in the url
Lenght limitation of 255 characters
Often cacheable
Support only strings data type

Settings saved in browser history
Result can be bookmarked

Faster but less security

POST

Values contained in message's body
No lenght limitation
Hardly cacheable
Supports different data types
